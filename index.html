1

¿Qué permite esta integración?

El protocolo MCP (Model Context Protocol) actúa como un puente directo entre el chat estándar de Claude Desktop y tu archivo abierto en Power BI Desktop. En lugar de escribir código a mano o copiar y pegar, Claude se conecta localmente a tu modelo para crear tablas, medidas DAX y establecer relaciones de forma automática.

Ejemplo de uso en un inicio

"Con tu informe abierto, le dices a Claude desde el chat: Vamos a empezar un nuevo Dashboard usando una metodología en el archivo que tengo abierto de PowerBI, así que, conéctate a el."

Claude recibe la instrucción y se conecta automáticamente al archivo de PowerBI

Ejemplo de uso en un proyecto real

"Con tu informe abierto ya con las tablas de datos insertadas, le dices a Claude desde el chat: Crea una medida DAX que calcule una tendencia mostrando el crecimiento real de tickets por mes."

Claude recibe la instrucción, lee el esquema de tus tablas actuales en Power BI y resuelve directamente en tu modelo sin que tengas que tocar el editor avanzado.

2

Prerrequisitos de software

Para establecer este puente de comunicación, necesitaremos instalar dos programas base. Las instalaciones pueden hacerse con las opciones por defecto.

1

Claude Desktop

Descarga la aplicación nativa de escritorio desde la página oficial de Anthropic. Nota importante: Aunque la versión gratuita funciona, es sumamente recomendable utilizar la versión de pago (Claude Pro), ya que la versión gratuita tiene límites de créditos estrictos que se consumirán rápidamente al enviarle esquemas completos de bases de datos a la IA.

2

Visual Studio Code (VS Code)

Descarga el editor de código gratuito de Microsoft. Ojo: No vamos a programar en VS Code ni le daremos instrucciones desde allí. Solo lo utilizaremos como vehículo para instalar la extensión del servidor MCP en nuestra computadora.

3

Instalación y extracción del Servidor MCP

1

Instalar la extensión en VS Code

Abre Visual Studio Code y dirígete al panel izquierdo en el ícono de Extensiones (o presiona Ctrl+Shift+X).

En el buscador escribe MCP. Busca la extensión Power BI Modeling MCP y asegúrate de que el proveedor oficial sea Microsoft. Haz clic en instalar. Una vez que termine, puedes cerrar VS Code completamente.

2

Extraer la ruta del servidor

Abre tu explorador de archivos en Windows. En la barra superior de búsqueda, escribe exactamente lo siguiente y presiona Enter:

%userprofile%

Navega a través de las siguientes carpetas:

  1. Abre la carpeta .vscode
  2. Entra a extensions
  3. Busca la carpeta de la extensión instalada (suele llamarse analysis-services.powerbi-modeling-mcp...)
  4. Entra a server

Allí encontrarás un archivo ejecutable llamado PowerBI_Modeling_MCP (o un nombre similar dependiendo de la versión). Haz clic derecho sobre él y selecciona la opción "Copiar como ruta de acceso".

4

Configuración en Claude Desktop

Ahora le indicaremos a Claude dónde se encuentra este servidor para que pueda ejecutarlo y conectarse a Power BI.

1

Abrir el archivo de Ajustes

Abre Claude Desktop. Ve a tu perfil en la esquina superior y selecciona el menú de Configuración.

En el panel izquierdo, baja hasta la sección que dice Desarrollador. Haz clic en el botón para editar la configuración de MCP. Esto abrirá un archivo claude_desktop_config.json en tu Bloc de Notas.

2

Inyectar el código y formatear la ruta

Reemplaza el contenido del Bloc de Notas con la estructura base del MCP de Power BI, pero deberás pegar la ruta que copiaste en el paso anterior en la propiedad "command". Tu archivo debería quedar similar a esto:

{
  "mcpServers": {
    "powerbi-modeling-mcp": {
      "command": "C:\\Users\\TuUsuario\\.vscode\\extensions\\analysis-services.powerbi-modeling-mcp-0.4.0-win32-x64\\server\\PowerBI_Modeling_MCP.exe",
      "args": ["--start"],
      "env": {}
    }
  }
}
Importante: Las dobles barras
Cuando copies tu ruta, Windows usará barras simples (\). Para que el archivo JSON sea válido, debes presionar Ctrl+H en el Bloc de Notas y reemplazar todas las barras simples (\) por barras dobles (\\). Guarda los cambios en el archivo y ciérralo.
3

Reiniciar y Validar

Es indispensable reiniciar la aplicación de Claude. No basta con cerrar la ventana. Ve a la barra de tareas de Windows (cerca del reloj), haz clic derecho en el ícono de Claude y selecciona Salir.

Vuelve a abrir Claude Desktop, entra de nuevo a Configuración > Desarrollador y revisa el estatus. Debería indicar que el servidor MCP se ha conectado y está corriendo exitosamente.

Tecnologías de apoyo

1

Power BI MCP

Conexión directa al modelo semántico abierto. Claude puede leer tablas, medidas, relaciones y crear/modificar medidas DAX sin intervención manual.

2

Deneb (Vega-Lite)

Motor de visualización declarativa que permite gráficas imposibles con visuals nativos: degradados, anotaciones condicionales, microcharts, layouts custom.

3

HTML Content

Tarjetas KPI, tablas con formato avanzado e iconografía dinámica construidas con HTML/CSS puro, reactivas al contexto de filtro de Power BI.

4

Filtro General Universal

Visual personalizado TypeScript que genera una barra de filtros interactiva en la parte superior del dashboard. Soporta búsqueda por texto, multi-selección, meses ordenados en español y color de acento configurable para cualquier paleta.

Para instalarlo en Power BI Desktop:

  1. Descarga el archivo FiltroGeneralUniversal.pbiviz desde el repositorio: Descargar Filtro
  2. En PBI Desktop → Inicio → Más objetos visuales → Importar desde archivo
  3. Selecciona el archivo descargado y acepta la advertencia de certificación
  4. El visual aparece en tu panel de Visualizaciones listo para usar
Uso del filtro
Cuando le pidas a Claude que agregue un filtro a tu dashboard, la skill powerbi-filter le dará instrucciones específicas sobre qué columnas poner (Año, Mes, dimensiones), el DAX necesario para ordenar los meses y cómo configurar el visual en tu lienzo.

Las 7 Etapas del Flujo de Trabajo

El flujo está dividido en 7 etapas. Las etapas 1-5 son colaborativas (Claude + Analista). Las etapas 6-7 son del Analista con Claude en modo soporte.

1

Ingesta y diagnóstico de datos

Claude realiza un análisis automático antes de cualquier construcción. Revisa tipos de columnas, nulos, duplicados y outliers. Identifica dimensiones candidatas y hechos. Por último, señala problemas de calidad que deben resolverse antes de modelar.

2

Intención del dashboard

Esta es la etapa más importante del flujo. Lo que el analista declare como objetivo del dashboard es mandatorio. Claude acata la intención y construye todo en función de ella.

3

Diseño del modelo semántico

El modelo se diseña para servir los KPIs mandatorios de la Etapa 2. Claude se conecta al modelo abierto en Power BI Desktop para confirmar que el esquema coincide con lo diseñado y detectar particiones sin datos.

4

Construcción de medidas DAX

Claude escribe y sube las medidas directamente al modelo abierto via MCP. Las medidas se construyen en el orden dictado por la intención declarada en Etapa 2.

5

Diseño de visuales y estilo

El diseño sigue una jerarquía clara: primero la intención del analista, luego el criterio de Claude.

6

Layout y publicación

Esta etapa queda a cargo del analista. Claude entrega una guía de layout específica para el dashboard construido, no genérica.

7

Revisión y correcciones

Una vez que el analista tiene el dashboard armado en el canvas, Claude queda en modo de soporte activo para correcciones.

8

Prompt de Inicio

Con la configuración lista, usa este prompt al comenzar cada nuevo dashboard. Copia el bloque, personaliza los campos entre corchetes y pégalo directamente en Claude Desktop. Esto activa automáticamente el flujo de 7 etapas.

Prompt de inicio — copiar y personalizar
Quiero construir un dashboard de Power BI con la Metodología 1 (MCP + Chat).

Mi archivo Power BI está abierto: [nombre del archivo.pbix]
Tablas disponibles: [describe las tablas y columnas principales]
Objetivo del dashboard: [qué preguntas debe responder / quién lo usará]
Paleta de colores: [morada / azul / verde / naranja / hex corporativo]

Conéctate al modelo y comenzamos con la Etapa 1.
¿Qué hace Claude con este prompt?
  1. Activa la skill powerbi-mcp-flow y se conecta al modelo abierto via MCP
  2. Lee las tablas y columnas para diagnosticar la calidad de los datos (Etapa 1)
  3. Te pregunta o confirma el objetivo del dashboard (Etapa 2)
  4. Crea las medidas DAX necesarias directamente en tu modelo (Etapa 4)
  5. Genera el código de cada visual y te da instrucciones exactas de dónde colocarlo (Etapa 5–6)
9

Checklist de Sesión

Usar este checklist al iniciar cada nuevo proyecto de dashboard:

  • Datos recibidos: CSV, Excel o conexión directa a BD disponible.
  • Power BI Desktop abierto: Archivo.pbix abierto para conexión MCP.
  • Intención del dashboard declarada: El analista definió el objetivo principal y los KPIs mandatorios.
  • Referencia de diseño (opcional): Captura, paleta o descripción de estilo. Si no se da, Claude decide.
  • Audiencia del dashboard: ¿Quién lo usa? ¿Con qué frecuencia?
  • Fuente de datos confirmada: Ruta local o conexión a servidor validada.
  • Granularidad temporal definida: ¿Diario, semanal, mensual?
  • Fechas del período de análisis: Rango histórico disponible.
  • Etapa 7 activada (si aplica): El analista puede pedir correcciones en cualquier momento después del layout.

1

¿Qué es este sistema?

Este sistema permite que Claude construya dashboards completos de Power BI automáticamente a partir de una descripción en lenguaje natural. Tú describes lo que necesitas; Claude diseña el modelo, crea las medidas DAX, genera los visuales y arma el layout.

Cómo funciona

El pipeline lo llevan las skills de Microsoft Fabric (planifican, diseñan, posicionan en una cuadrícula, cargan datos y arman la navegación entre páginas), y los visuales que se dibujan son custom visuals: gráficas con Deneb, tarjetas con HTML Content y filtros con el visual Filtro — en lugar de los visuales nativos de Power BI.

Las dos capas del sistema

CapaHerramientaHacePower BI
ModeloMCP de Power BI (incluido en el plugin de Fabric)Crear tablas, medidas DAX (incluidas las HTML) y relacionesAbierto
Visuales y layoutSkills de Fabric + generador hybrid.js + 2 CLIs de FabricEscribir los visuales (Deneb/HTML/Filtro) en la cuadrícula, validar, recargar y capturarAbierto (se refresca con reload)

Tipos de visuales que construye

  • Gráficas con Deneb (Vega-Lite): interactivas; al hacer clic en una barra, el resto de gráficas y tarjetas se filtran.
  • Tarjetas KPI con HTML Content: una medida DAX que devuelve HTML, con badge de cambio vs. período anterior.
  • Barra de filtros con el visual Filtro: pills con búsqueda, multi-select y color coordinado con el dashboard.
  • Navegación entre páginas: navegador de páginas que arma la skill de Fabric.

Las skills de Fabric se instalan aparte
El pipeline (planificación, diseño, posicionamiento, navegación, validación) lo aportan las skills de Microsoft Fabric, que se instalan como plugin desde su marketplace público (Sección 3). Ese mismo plugin trae el MCP de modelado. Las skills de la carpeta skills/ son las de los patrones de Deneb, las tarjetas HTML y el Filtro.
3

Instalación

El script setup.ps1 hace casi todo: verifica/instala Node.js, instala las 2 CLIs de Fabric, copia el motor y el visual Filtro, y registra el marketplace de skills de Fabric.

1

Ejecutar el instalador

Abre PowerShell en la carpeta EXPORTAR y ejecuta:

PowerShell -ExecutionPolicy Bypass -File setup.ps1

El script imprime cada paso con OK / ! / X. Los "!" son avisos (por ejemplo, "reinicia la terminal para ver la CLI recién instalada"), no errores.

2

Instalar el plugin de skills de Fabric (en Claude Code)

En una sesión de Claude Code escribe:

/plugin marketplace add microsoft/skills-for-fabric
/plugin install powerbi-authoring@fabric-collection

Ese plugin trae las skills de autoría de reportes y el MCP de modelado de Power BI (se ejecuta con npx). Reinicia Claude Code al terminar.

3

Power BI Desktop

Instálalo desde la Microsoft Store o powerbi.microsoft.com/desktop y activa el formato de proyecto: Archivo → Opciones → Características de vista previa → Power BI Project (.pbip). El sistema trabaja sobre archivos .pbip (texto), no .pbix.

Y ya
Con eso queda instalado. Lo único que se repite por cada reporte nuevo es importar los tres custom visuals (Sección 4).
4

Importar los custom visuals en Power BI

Los custom visuals no viajan dentro del .pbip: se importan una vez por reporte. Son los que reemplazan a los visuales nativos.

1

Deneb y HTML Content — desde AppSource

Con el reporte abierto: Inicio → Más objetos visuales → Desde AppSource. Busca y agrega Deneb (gráficas) y HTML Content (de Nova Silva / Daniel Marsh-Patrick, para las tarjetas).

2

Filtro — desde archivo

Inicio → Más objetos visuales → Importar desde archivo → selecciona FiltroGeneralUniversal.pbiviz. Acepta el aviso de seguridad → "Importación correcta".

3

Guardar

Ctrl+S para que los visuales queden registrados en el reporte. Una vez importados, permanecen ahí.

5

Verificar que todo funciona

1

Node y las CLIs de Fabric

node --version
powerbi-report-author --help
powerbi-desktop status

Los tres deben responder. Si una CLI "no se reconoce", cierra y reabre la terminal.

2

Las skills y el MCP en Claude Code

Con Power BI Desktop abierto en un reporte, en Claude Code escribe:

Conéctate al modelo de Power BI que tengo abierto y lista las tablas

Claude debería conectarse por el MCP y describir el modelo. Si no, revisa que el plugin powerbi-authoring esté instalado y que Node esté en el PATH.

6

Construir tu primer dashboard

Con el sistema instalado y los custom visuals importados, construir un dashboard es básicamente un prompt. Le describes a Claude qué datos tienes y qué quieres ver, y él se encarga del resto.

Prompt de inicio (copiar y adaptar)
Vamos a construir un dashboard de Power BI
(skills de Fabric + Deneb / HTML Content / Filtro).

Archivo: C:\ruta\a\MiReporte.pbip
Datos: [tablas y campos clave, ej. "viajes con Fecha, Estación, Tipo de usuario, Duración"]
Objetivo: [qué debe responder, ej. "viajes por mes, top estaciones y % miembros vs casuales"]
Páginas: [ej. Resumen, Patrones, Estaciones]
Paleta: [color de acento en hex o nombre]

A partir de ahí no tienes que hacer nada más: Claude ejecuta todo el flujo por su cuenta e itera hasta que quede bien. Por dentro:

  • Conecta el modelo y crea las medidas DAX (y las medidas HTML de las tarjetas).
  • Diseña el layout y las páginas con las skills de Fabric.
  • Genera los visuales, valida el reporte y recarga Power BI.
  • Te muestra capturas del resultado y corrige lo que haga falta.

Cuando lo veas, pides ajustes en lenguaje natural ("haz la gráfica de barras horizontal", "cambia el acento a azul") y Claude los aplica.

Con muchos datos, carga los datos tú (no por el MCP)
Si el dataset es grande, haz el Actualizar de los datos manualmente en Power BI Desktop, no le pidas a Claude que refresque por el MCP: cargar volúmenes grandes por el MCP consume mucha RAM y suele hacer timeout. Claude arma el modelo, las medidas y los visuales; tú aprietas Actualizar para traer los datos pesados y guardas con Ctrl+S.
