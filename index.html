¿Qué permite esta metodología?

El protocolo MCP (Model Context Protocol) actúa como un puente directo entre el chat estándar de Claude Desktop y tu archivo abierto en Power BI Desktop. Claude se conecta localmente a tu modelo para crear medidas DAX, leer el esquema y generar specs de visuales, sin que tengas que tocar el editor avanzado de PBI.

Cómo funciona en la práctica

Con tu reporte abierto en Power BI, le dices a Claude: "Crea una medida DAX que calcule el promedio de tickets por agente para el mes seleccionado."

Claude lee el esquema de tus tablas actuales vía MCP y escribe la medida directamente en tu modelo, sin que tengas que abrir el editor avanzado.

Diferencia con Metodología 2

En M1, tú colocas los visuales manualmente en el canvas de Power BI. Claude se encarga de la lógica (DAX), entrega el JSON de las gráficas Deneb para que lo pegues, y puede compilar el filtro personalizado.

En M2 (Motor Automatizado), Claude también coloca los visuales automáticamente mediante scripts Python.

Lo que Claude hace por ti en M1

Lo que tú haces

  • Insertar los visuales en el canvas (Deneb, HTML Content, Filtro)
  • Arrastrar medidas al campo "Values" de cada visual
  • Pegar el JSON de Deneb en "Edit specification"
  • Ajustar el layout y tamaños a tu gusto

Instalación del Servidor MCP

1

Instalar la extensión en VS Code

Abre Visual Studio Code → panel de Extensiones (Ctrl+Shift+X). Busca Power BI Modeling MCP y asegúrate de que el proveedor sea Microsoft. Haz clic en Instalar. Al terminar, puedes cerrar VS Code.

2

Encontrar la ruta del ejecutable

Abre PowerShell y ejecuta:

Get-ChildItem "$env:USERPROFILE\.vscode\extensions" | Where-Object { $_.Name -like "*powerbi*" }

Verás algo como analysis-services.powerbi-modeling-mcp-0.4.0-win32-x64. Anota ese nombre de carpeta.

El ejecutable está dentro de esa carpeta en la subcarpeta server\. Para copiar la ruta exacta, navega en el Explorador de archivos a %userprofile%\.vscode\extensions\[carpeta]\server\, haz clic derecho sobre el archivo .exe y selecciona "Copiar como ruta de acceso".

Tecnologías de apoyo

1

Power BI MCP

Conexión directa al modelo semántico abierto en Power BI Desktop. Claude puede leer tablas, medidas y relaciones, y crear o modificar medidas DAX sin intervención manual.

2

Deneb (Vega-Lite)

Motor de visualización declarativa que permite gráficas imposibles con visuals nativos: degradados, anotaciones condicionales, cross-filter al hacer clic, hover interactivo. Claude genera el JSON completo; tú lo pegas en el editor de Deneb.

3

HTML Content

Tarjetas KPI con diseño a medida, badges de cambio vs período anterior y hover animado. Se construyen como medidas DAX que devuelven HTML/CSS. Claude las crea directamente en tu modelo vía MCP.

Las 7 Etapas del Flujo de Trabajo

Las etapas 1–5 son colaborativas (Claude + tú). La etapa 6 (layout) es tuya, con Claude en modo soporte. La etapa 7 es de revisión iterativa.

1

Ingesta y diagnóstico de datos

Claude se conecta al modelo vía MCP y realiza un análisis automático: tipos de columnas, nulos, duplicados, dimensiones candidatas y hechos. Señala problemas de calidad antes de modelar.

2

Intención del dashboard

Tú declaras el objetivo del dashboard y los KPIs mandatorios. Esta es la etapa más importante: todo lo que Claude construya gira en torno a esta intención declarada.

3

Diseño del modelo semántico

Claude revisa el esquema del modelo abierto en PBI para confirmar que tiene lo necesario para los KPIs de la Etapa 2. Detecta tablas faltantes o relaciones incorrectas.

4

Construcción de medidas DAX

Claude escribe y sube las medidas directamente al modelo vía MCP. Las crea en orden: primero las base (conteos, sumas), luego variaciones de período, luego las medidas HTML para tarjetas KPI.

5

Generación de visuales

Claude entrega el JSON completo de cada gráfica Deneb (listo para pegar) y puede compilar e instalar el filtro personalizado. Tú insertas los visuals en el canvas y pegas los specs.

6

Layout y publicación

Esta etapa es tuya. Claude entrega una guía de layout específica: qué visual va dónde, tamaños recomendados, orden de capas. Tú armas el canvas en PBI.

7

Revisión y correcciones

Con el dashboard armado, Claude queda en modo soporte activo: ajusta colores, corrige medidas, modifica el JSON de Deneb. Puedes iterar las veces que necesites.

7

Skills Instalables para Claude

Las skills son archivos de instrucciones que le enseñan a Claude cuándo y cómo comportarse. Al instalarlas, Claude activa automáticamente el flujo correcto sin que tengas que explicarle cada vez.

SkillDescripción
powerbi-mcp-flow Orquestador principal de M1. Activa el flujo de 7 etapas, conecta al modelo vía MCP y genera instrucciones de colocación para cada visual.
deneb-chart-generator Genera specs Vega-Lite completas para Deneb: barras, líneas, donut. Con cross-filter y hover interactivo incorporados.
html-kpi-cards Construye medidas DAX de tipo HTML Content para tarjetas KPI con diseño, hover, badge de cambio vs período y tooltip.
powerbi-dax-auditor Audita el modelo en busca de blanks, medidas huérfanas y problemas de cardinalidad. Propone correcciones.

Cómo instalar las skills

1

Abrir el menú Customize en Claude Desktop

Abre Claude Desktop y ve al menú principal. Haz clic en la opción Customize (Personalizar).

2

Crear nueva habilidad

En el panel izquierdo de Customize, haz clic en el ícono + junto a la barra de búsqueda de Habilidades. Selecciona Crear habilidad → Subir una habilidad.

3

Cargar el archivo SKILL.md

Arrastra y suelta el archivo SKILL.md de la skill que quieres instalar, o navega hasta él. Repite para cada una de las 4 skills del paquete M1.

Alternativa: instalación por carpeta

Desde PowerShell, encuentra la carpeta de skills de Claude Desktop:

Get-ChildItem "$env:APPDATA\Claude\local-agent-mode-sessions\skills-plugin" -Recurse -Filter "skills" -Directory

Copia las carpetas de skills a esa ruta y reinicia Claude Desktop.

¿Cuándo se activan las skills?
Las skills se activan automáticamente cuando Claude detecta el contexto correcto. Mencionar "dashboard", "gráfica" o "tarjeta KPI" basta para que Claude sepa qué hacer.

Checklist de Sesión

Usa este checklist al iniciar cada nuevo proyecto de dashboard:

  • Datos cargados: El archivo .pbip o .pbix tiene las tablas importadas y el modelo listo.
  • Power BI Desktop abierto con el reporte activo (necesario para la conexión MCP).
  • Intención declarada: Define el objetivo del dashboard y los KPIs que debe responder.
  • Deneb y HTML Content importados en el reporte (una vez por reporte nuevo, desde AppSource).
  • Filtro importado (si usas el visual de filtros): Inicio → Más objetos visuales → Importar desde archivo → seleccionar el .pbiviz.
  • Paleta de colores: Un hex de acento principal (ej. #6A0DAD). Si no se define, Claude elige.
  • Granularidad temporal: ¿El dashboard filtra por día, semana, mes?
  • Audiencia: ¿Quién usará el dashboard? ¿Con qué frecuencia?
Para empezar

Con todo lo anterior listo, abre una sesión en Claude Desktop y escribe:

Vamos a empezar un dashboard nuevo usando la metodologia MCP + Chat.
Tengo un archivo de Power BI abierto. Conectate al modelo.

Claude activará el flujo, se conectará al modelo y te guiará paso a paso por las 7 etapas.

¿Qué es este sistema?

Este sistema permite que Claude construya dashboards completos de Power BI automáticamente a partir de una descripción en lenguaje natural. Tú describes lo que necesitas; Claude diseña el layout, crea todas las medidas DAX, genera las gráficas interactivas y coloca los visuales en las posiciones exactas dentro del archivo del reporte.

Ejemplo de uso

Le dices a Claude: "Tengo datos de soporte técnico. Quiero un dashboard con los tickets por categoría, la tendencia mensual y los tiempos de resolución. Paleta azul corporativo."

Claude hace todo: crea las medidas DAX vía MCP, diseña el layout en cuadrícula, genera las gráficas Deneb interactivas, configura el filtro y escribe todos los visuales en los archivos del reporte. Abres PBI y el dashboard está listo.

Las dos capas del sistema

CapaHerramientaHacePBI debe estar
ModeloMCP Power BICrear medidas DAX, tablas, relacionesABIERTO
VisualesScripts Python (motor)Colocar gráficas, tarjetas, filtros, layoutCERRADO
Regla de oro
El motor Python edita los archivos del reporte directamente. Si Power BI está abierto al mismo tiempo, los cambios se pierden. El wrapper pbi.ps1 gestiona esto automáticamente: cierra PBI, edita, vuelve a abrirlo.

Tipos de visuales que el sistema construye

  • Tarjetas KPI — HTML Content: diseño profesional con hover y badge de cambio vs período anterior
  • Gráficas — Deneb (Vega-Lite): al hacer clic en una barra, todas las demás gráficas y tarjetas se filtran
  • Barra de filtros — visual personalizado Filtro: pills con búsqueda, multi-select y color coordinado
  • Botones de navegación — botones nativos de PBI con bookmarks automáticos

Iniciar el primer proyecto

Una vez verificado todo, aquí está el flujo completo para iniciar un dashboard desde cero.

1

Tener un archivo .pbip con los datos cargados

Necesitas un archivo en formato .pbip (no .pbix). Para convertir: en PBI Desktop ve a Archivo → Guardar como → formato "Power BI Project (.pbip)".

Los datos ya deben estar cargados. Claude puede crear las medidas DAX, pero los datos fuente deben estar.

2

Registrar el proyecto en el motor

cd "C:\Claude\pbi-motor"
.\pbi.ps1 pbi-init "C:\ruta\a\MiReporte.pbip"

El motor detecta las carpetas del reporte y escribe un pbi.config.json. Todos los comandos operarán sobre este proyecto a partir de aquí.

3

Detectar los GUIDs de los custom visuals

.\pbi.ps1 detect-custom

Devuelve los identificadores únicos de Deneb y HTML Content instalados en este reporte específico. Los GUIDs son distintos por versión e instalación.

Nunca reutilices GUIDs de otro reporte
Los GUIDs cambian por proyecto. Si usas los de un reporte anterior, Claude generará visuals vacíos. Siempre ejecuta detect-custom al iniciar un proyecto nuevo.
4

Importar el visual Filtro en Power BI (una sola vez por reporte)

  1. Abre el reporte en Power BI Desktop
  2. Ve a Inicio → Más objetos visuales → Importar desde archivo
  3. Navega al archivo .pbiviz del Filtro
  4. Acepta el aviso de seguridad → debe aparecer "Importación correcta"
Solo una vez por reporte
Una vez importado, el visual queda disponible en ese reporte permanentemente.
5

Decirle a Claude que empiece

En Claude Desktop, abre una sesión nueva con tu carpeta del motor como contexto y escribe:

vamos a empezar un dashboard nuevo

Claude activará el skill de construcción de dashboards, se conectará al modelo, diseñará el layout y construirá todos los visuales con:

.\pbi.ps1 pbi-refresh build-dashboard --spec mi_dashboard.json

Este comando cierra PBI, escribe todos los visuales en los archivos del reporte, y vuelve a abrir PBI con el dashboard listo.

Solución de problemas frecuentes

SíntomaCausa probableSolución
py: command not found Python no se agregó al PATH Reinstalar Python marcando "Add Python to PATH"
"la ejecución de scripts está deshabilitada" Restricción de PowerShell por defecto Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
El MCP no conecta a Power BI Ruta incorrecta en config o PBI no está abierto Verificar ruta del ejecutable; asegurarse de que PBI tiene un reporte abierto
Visual Deneb vacío en PBI GUIDs del proyecto anterior reutilizados Correr .\pbi.ps1 detect-custom para obtener los GUIDs reales de este reporte
Filtro aparece vacío / "importa el visual" .pbiviz no importado en este reporte PBI → Inicio → Más objetos visuales → Importar desde archivo → seleccionar el .pbiviz
Claude no activa el modo dashboard Skills no instaladas o carpeta incorrecta Verificar que los SKILL.md están en la carpeta correcta y reiniciar Claude Desktop
"Página 'X' no encontrada" Nombre de página no coincide exactamente .\pbi.ps1 list-pages y copiar el nombre exacto (diferencia mayúsculas)
Los cambios se pierden al abrir PBI PBI estaba abierto cuando corrió build-dashboard Siempre usar pbi-refresh en lugar de build-dashboard directo
Espacio vacío al fondo de una tarjeta HTML Falta container-type: size en el body tag Agregar style='container-type: size;' al tag <body> en la medida DAX
Error "ModuleNotFoundError: pbir_layout" Los 3 archivos Python no están juntos Verificar que pbir_tool.py, pbir_templates.py y pbir_layout.py están en la misma carpeta
Ciclo de trabajo normal
  1. Abre Power BI → crea/edita medidas con Claude vía MCP (PBI abierto)
  2. Claude cierra PBI automáticamente, construye los visuales y vuelve a abrir PBI
  3. Revisas en PBI, describes los ajustes → Claude los hace
  4. Repites hasta que el dashboard quede bien

Si algo sale mal: .\pbi.ps1 validate-page --page "Nombre" detecta el problema; fix-page lo corrige automáticamente.
