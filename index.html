1

¿Qué es este sistema?

Este sistema permite que Claude Code cree dashboards completos de Power BI automáticamente a partir de una descripción en lenguaje natural. Tú describes lo que necesitas; Claude diseña, construye y coloca todos los visuales.

Ejemplo de uso

Le dices a Claude: "Tengo datos de soporte técnico. Quiero un dashboard con los tickets por categoría, la tendencia mensual, y los tiempos de resolución. Paleta azul corporativo."

Claude hace todo: crea las medidas DAX, diseña el layout, genera las gráficas Deneb interactivas, configura el filtro y construye el dashboard.

Las dos capas del sistema

Capa Herramienta Hace Power BI debe estar
Modelo MCP Power BI Crear medidas DAX, tablas, relaciones ABIERTO
Visuales Scripts Python (motor) Colocar gráficas, tarjetas, filtros, layout CERRADO
Regla de oro
El motor Python edita los archivos del reporte directamente — si Power BI está abierto al mismo tiempo, los cambios se pierden. El wrapper pbi.ps1 gestiona esto automáticamente: cierra PBI, edita, vuelve a abrirlo.

Tipos de visuales que el sistema construye

  • Tarjetas KPI — usando HTML Content: bonitas, con hover y badge de cambio vs período anterior
  • Gráficas — usando Deneb (Vega-Lite): interactivas, al hacer clic en una barra, todas las demás gráficas y tarjetas se filtran
  • Barra de filtros — usando el visual personalizado Navara: pills con búsqueda, multi-select, y color coordinado con el dashboard
  • Botones de navegación — usando botones nativos de Power BI

Archivos incluidos en este paquete

La carpeta EXPORTAR que acompaña este tutorial tiene esta estructura:

    powerbi-dashboard-builder/
      SKILL.md          ← orquestador principal
    deneb-chart-generator/

¿Qué son los SKILL.md?
Los archivos SKILL.md son instrucciones que le dicen a Claude cómo comportarse en diferentes situaciones. Al instalarlos, Claude sabe automáticamente cuándo activar cada modo (construir gráficas, crear tarjetas, auditar el modelo). No son código que se ejecuta — son guías de comportamiento.
3

Instalar los prerrequisitos de software

Orden importante
Instala en el orden indicado. VS Code debe instalarse antes de la extensión del MCP.
1

Python 3 — el motor de scripts

Ve a python.org/downloads y descarga la versión más reciente de Python 3 para Windows.

Durante la instalación, marca obligatoriamente la casilla "Add Python to PATH" (está en la primera pantalla del instalador). Sin esto, el sistema no funcionará.

Para verificar que quedó bien instalado, abre PowerShell y escribe:

py --version

Debe aparecer algo como Python 3.14.x. Si aparece un error, reinstala marcando el checkbox de PATH.

Nota sobre el comando
En este sistema siempre se usa py (el launcher de Windows), nunca python ni python3. Es importante para que funcionen los scripts.
2

Power BI Desktop — la aplicación principal

Descárgalo desde la Microsoft Store (busca "Power BI Desktop") o desde powerbi.microsoft.com/desktop.

Una vez instalado, abre Power BI Desktop y sigue estos pasos para instalar los dos custom visuals que el sistema necesita:

  1. En la cinta superior, haz clic en Inicio
  2. Busca el grupo Insertar → haz clic en "Más objetos visuales"
  3. Elige "Desde AppSource"
  4. Busca "Deneb" → haz clic en Agregar
  5. Repite con "HTML Content" (de Nova Silva / Daniel Marsh-Patrick)
Paso obligatorio
Deneb y HTML Content deben estar instalados en cada reporte donde vayas a trabajar. Son los visuals que Claude usa para gráficas y tarjetas KPI. Sin ellos el sistema genera visuals vacíos.
3

VS Code — solo para el MCP de Power BI

Ve a code.visualstudio.com y descarga VS Code para Windows. La instalación es estándar (siguiente, siguiente, instalar).

Una vez instalado VS Code, instala la extensión del MCP de Power BI:

  1. Abre VS Code
  2. En el panel izquierdo, haz clic en el ícono de Extensiones (cuadrado con 4 piezas)
  3. En el buscador escribe: Power BI Modeling MCP
  4. El resultado debería ser de "Microsoft" / "Analysis Services" → haz clic en Instalar

No necesitas abrir VS Code para nada más — solo fue para instalar la extensión.

¿Por qué VS Code?
La extensión "Power BI Modeling MCP" de Microsoft solo se distribuye a través del marketplace de VS Code. Una vez instalada, la extensión genera un ejecutable que Claude usa para comunicarse con Power BI. VS Code en sí no se necesita abierto durante el trabajo.
4

Claude Desktop — la app de Anthropic

Descarga e instala Claude Desktop desde claude.ai. Entra con tu cuenta de Claude.

Esta es la aplicación principal donde trabajarás con el sistema.

4

Copiar los archivos del motor a tu máquina

Los archivos del motor son los scripts Python y PowerShell que Claude usa para construir los dashboards. Necesitas copiarlos a una carpeta fija en tu máquina.

1

Crear la carpeta del motor

Crea una carpeta en tu máquina donde vivirán todos los archivos del sistema. La ruta puede ser cualquiera, pero debe ser una ruta sin espacios ni caracteres especiales. Recomendamos:

C:\Claude\pbi-motor\

Para crearla, abre PowerShell y escribe:

New-Item -ItemType Directory -Force "C:\Claude\pbi-motor"
2

Copiar los archivos del motor

De la carpeta EXPORTAR\motor\ que recibes con este tutorial, copia todos los archivos a C:\Claude\pbi-motor\:

  • pbir_tool.py
  • pbir_templates.py
  • pbir_layout.py
  • pbi.ps1
  • SISTEMA.md
  • filtros_test.json (ejemplo, opcional)

Al terminar, C:\Claude\pbi-motor\ debe tener exactamente esos 6 archivos.

3

Actualizar la ruta en SISTEMA.md y el SKILL.md principal

El archivo SISTEMA.md y el skill powerbi-dashboard-builder tienen una ruta que apunta a la máquina original. Necesitas actualizarla a tu nueva ruta.

Abre C:\Claude\pbi-motor\SISTEMA.md con el Bloc de notas (clic derecho → Abrir con → Bloc de notas).

Usa Ctrl+H (buscar y reemplazar) y reemplaza:

C:\Users\<usuario-origen>\AppData\Roaming\Claude\...

por:

C:\Claude\pbi-motor

Guarda el archivo. Repite lo mismo en el archivo SKILL.md de powerbi-dashboard-builder (lo instalarás en la siguiente sección, pero hazlo antes de copiarlo).

¿Por qué esto?
Claude lee SISTEMA.md al inicio de cada sesión de trabajo para saber cómo usar los scripts. Si la ruta está mal, Claude no encontrará los archivos y no podrá ejecutar comandos.
4

Copiar el filtro Navara

De la carpeta EXPORTAR\navara\, copia el archivo .pbiviz a un lugar conocido. Puedes ponerlo dentro del motor:

C:\Claude\pbi-motor\navara\navarafilter...1.1.0.0.pbiviz

Este archivo es el visual personalizado de filtros. Se importa en cada reporte nuevo desde Power BI (lo verás en la Sección 8).

5

Configurar el MCP de Power BI en Claude

El MCP (Model Context Protocol) es la "línea de comunicación" entre Claude y Power BI. Permite que Claude cree medidas DAX, lea el modelo de datos y trabaje con el archivo directamente mientras Power BI está abierto.

1

Encontrar el ejecutable del MCP

Cuando instalaste la extensión en VS Code, esta dejó un ejecutable en una carpeta de extensiones. Para encontrarlo, abre PowerShell y escribe:

Get-ChildItem "$env:USERPROFILE\.vscode\extensions" | Where-Object { $_.Name -like "*powerbi*" }

Deberías ver algo como:

analysis-services.powerbi-modeling-mcp-0.4.0-win32-x64

Anota ese nombre completo de carpeta — lo necesitas en el siguiente paso.

2

Abrir el archivo de configuración de Claude Desktop

Claude Desktop usa un archivo JSON para saber qué MCPs cargar. Está en:

C:\Users\TU_USUARIO\AppData\Roaming\Claude\claude_desktop_config.json

Para abrirlo rápidamente, abre PowerShell y escribe:

notepad "$env:APPDATA\Claude\claude_desktop_config.json"

Si el archivo no existe, se abrirá el Bloc de notas con un archivo en blanco — está bien, lo crearás tú.

3

Agregar la configuración del MCP

Si el archivo está vacío, pega esto (reemplazando TU_USUARIO y el nombre de la carpeta de la extensión):

{
  "mcpServers": {
    "powerbi-modeling-mcp": {
      "command": "C:\\Users\\TU_USUARIO\\.vscode\\extensions\\analysis-services.powerbi-modeling-mcp-0.4.0-win32-x64\\server\\powerbi-modeling-mcp.exe",
      "args": ["--start"],
      "env": {}
    }
  }
}

Si el archivo ya tenía contenido (otras configuraciones), agrega solo el bloque "powerbi-modeling-mcp": {...} dentro de la sección "mcpServers" existente.

Cuidado con las barras
En JSON, las barras invertidas de las rutas de Windows deben escribirse dobles: C:\\Users\\... en lugar de C:\Users\.... Si pones solo una barra, el archivo JSON quedará inválido y Claude no cargará.
4

Reiniciar Claude Desktop

Cierra Claude Desktop completamente (clic derecho en el ícono del sistema → Salir) y vuelve a abrirlo. Al iniciar, Claude carga la configuración del MCP.

Para verificar que el MCP está activo, abre una sesión en Claude Desktop y escribe:

powerbi-modeling-mcp → connection_operations → ListLocalInstances

Si Power BI Desktop está abierto con un reporte, deberías ver la lista de instancias disponibles. Si Power BI no está abierto, recibirás una lista vacía — eso también es correcto.

6

Instalar las skills en Claude

Las skills son archivos SKILL.md que le enseñan a Claude cuándo y cómo comportarse. Hay 4 skills en el paquete. Necesitas instalarlas en la carpeta correcta de Claude Desktop.

Opción A — Copiar directamente (recomendado)

1

Encontrar tu carpeta de skills

La carpeta de skills tiene un ID único por instalación. Para encontrarla, abre PowerShell y escribe:

Get-ChildItem "$env:APPDATA\Claude\local-agent-mode-sessions\skills-plugin" -Recurse -Filter "skills" -Directory | Select-Object -ExpandProperty FullName

Debería aparecer una ruta como:

C:\Users\TU_USUARIO\AppData\Roaming\Claude\local-agent-mode-sessions\
  skills-plugin\XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX\
  YYYYYYYY-YYYY-YYYY-YYYY-YYYYYYYYYYYY\skills

Esa es la carpeta donde debes poner las skills. Anota la ruta completa.

Si la carpeta no existe
La carpeta de skills solo aparece después de que Claude Desktop carga al menos una vez el plugin de skills. Si no aparece, abre Claude Desktop, espera que cargue completamente, ciérralo y vuelve a ejecutar el comando.
2

Copiar las 4 carpetas de skills

De EXPORTAR\skills\, copia las 4 carpetas completas a la carpeta de skills que encontraste:

  • powerbi-dashboard-builder\SKILL.md
  • deneb-chart-generator\SKILL.md
  • html-kpi-cards\SKILL.md
  • powerbi-dax-auditor\SKILL.md

Cada carpeta va dentro de la carpeta skills\. La estructura final debe ser:

...skills-plugin\XXXXXXXX\YYYYYYYY\skills\
  powerbi-dashboard-builder\SKILL.md
  deneb-chart-generator\SKILL.md
  html-kpi-cards\SKILL.md
  powerbi-dax-auditor\SKILL.md
3

Reiniciar Claude Desktop

Cierra y reabre Claude Desktop. Las skills se cargan al iniciar la aplicación.

Para verificar que las skills están activas, en una sesión de Claude escribe:

vamos a empezar un dashboard nuevo

Claude debería responder activando el flujo de construcción de dashboard (pedir la ruta del archivo .pbip). Si solo responde genéricamente sin mencionar el flujo del motor, las skills no cargaron — revisa la ruta.

Opción B — Crear las skills desde Claude (alternativa)

Si no puedes encontrar la carpeta automáticamente, puedes pedirle a Claude que cree cada skill. En una sesión de Claude Desktop:

  1. Abre el archivo SKILL.md de una skill con el Bloc de notas
  2. Copia todo su contenido
  3. En Claude Desktop escribe: "Crea una skill con este contenido:" y pega el texto
  4. Repite para cada una de las 4 skills
¿Cuándo se activan las skills?
Las skills se activan automáticamente cuando Claude detecta el contexto correcto en la conversación. No tienes que invocarlas manualmente — basta con mencionar "dashboard", "gráfica", "tarjeta KPI" o similares y Claude sabrá qué hacer.
7

Verificar que todo funciona

Antes de usar el sistema con un reporte real, verifica que los scripts funcionan correctamente.

1

Verificar Python

Abre PowerShell, navega a la carpeta del motor y ejecuta:

cd "C:\Claude\pbi-motor"
py pbir_tool.py --help

Debe aparecer una lista de comandos disponibles (list-pages, build-dashboard, etc.). Si aparece un error de módulo, verifica que los 3 archivos Python estén en la misma carpeta.

2

Verificar el wrapper PowerShell

cd "C:\Claude\pbi-motor"
.\pbi.ps1 list-pages

Debe aparecer un mensaje amarillo diciendo que no hay proyecto activo (porque todavía no has registrado ninguno). Ese mensaje es correcto — significa que el script funciona.

Error de política de ejecución
Si ves el error "la ejecución de scripts está deshabilitada", ejecuta primero: 
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
Responde "S" cuando pregunte. Esto permite ejecutar scripts PowerShell locales.
3

Verificar el MCP (requiere Power BI abierto)

Abre Power BI Desktop con cualquier reporte. Luego en Claude Desktop, en una sesión nueva, intenta:

Conecta al modelo de Power BI que tengo abierto

Claude debería usar el MCP para listar las instancias de Power BI y conectarse. Si responde que no puede conectarse, revisa la ruta del ejecutable en claude_desktop_config.json.

8

Iniciar el primer proyecto

Una vez verificado todo, aquí está el flujo completo para iniciar un dashboard real desde cero.

1

Tener un archivo .pbip de Power BI con los datos cargados

Necesitas un archivo de Power BI en formato .pbip (no .pbix). Si tienes un .pbix, puedes convertirlo: en Power BI Desktop, ve a Archivo → Guardar como y elige el formato "Power BI Project (.pbip)".

Los datos ya deben estar cargados en el modelo (tablas, columnas, etc.). Claude puede ayudarte a crear las medidas DAX, pero los datos fuente deben estar.

2

Registrar el proyecto en el motor

Abre PowerShell, navega al motor y registra tu proyecto:

cd "C:\Claude\pbi-motor"
.\pbi.ps1 pbi-init "C:\ruta\a\MiReporte.pbip"

El motor detecta automáticamente las carpetas del reporte y escribe un archivo pbi.config.json. A partir de aquí, todos los comandos operan sobre este proyecto.

3

Detectar los GUIDs de los custom visuals

.\pbi.ps1 detect-custom

Esto devuelve los identificadores únicos de Deneb y HTML Content instalados en este reporte específico. Los GUIDs son distintos por versión e instalación.

Nunca reutilices GUIDs de otro reporte
Los GUIDs cambian por proyecto. Si usas los de un reporte anterior, Claude generará visuals vacíos que Power BI no reconoce. Siempre ejecuta detect-custom al iniciar un proyecto nuevo.
4

Importar el filtro Navara en Power BI (una sola vez por reporte)

El filtro personalizado Navara no viaja dentro del archivo .pbip — hay que importarlo manualmente una vez por reporte:

  1. Abre el reporte en Power BI Desktop
  2. Ve a Inicio → Más objetos visuales → Importar desde archivo
  3. Navega a donde pusiste el archivo .pbiviz (carpeta navara\)
  4. Selecciona el archivo navarafilter...1.1.0.0.pbiviz
  5. Acepta el aviso de seguridad → debe aparecer "Importación correcta"
Solo una vez por reporte
Una vez importado, el visual queda disponible en ese reporte permanentemente. No necesitas reimportarlo en cada sesión de trabajo — solo cuando empiezas a trabajar en un reporte nuevo.
5

Decirle a Claude que empiece

En Claude Desktop, abre una sesión nueva con tu carpeta del motor como contexto y escribe:

vamos a empezar un dashboard nuevo

Claude activará automáticamente el skill de construcción de dashboards y te guiará a través del proceso: qué datos tienes, cuál es la intención del dashboard, qué paleta de colores usar, etc.

Desde ahí Claude hace el trabajo: crea medidas DAX via MCP (con Power BI abierto), diseña el layout, y construye los visuales con:

.\pbi.ps1 pbi-refresh build-dashboard --spec mi_dashboard.json

Este comando cierra Power BI, escribe todos los visuales en los archivos del reporte, y vuelve a abrir Power BI con el dashboard listo.

9

Referencia rápida de comandos

Todos los comandos se ejecutan desde la carpeta del motor con .\pbi.ps1 <comando>.

Comandos de proyecto

ComandoQué hace
pbi-init "C:\ruta\Reporte.pbip"Registra un proyecto nuevo. Autodetecta las carpetas del reporte.
pbi-closeCierra Power BI Desktop (espera a que cierre).
pbi-openAbre el archivo .pbip del proyecto activo.
pbi-refresh build-dashboard --spec X.jsonCierra PBI, construye el dashboard, reabre PBI.

Comandos de inspección

ComandoQué hace
list-pagesLista todas las páginas del reporte con sus IDs.
list-visuals --page "Nombre"Lista los visuals de una página con su posición y tipo.
detect-customDevuelve los GUIDs de Deneb y HTML Content del proyecto.
validate-page --page "Nombre"Lint: detecta propiedades inválidas que PBI rechazaría.
fix-page --page "Nombre"Corrige automáticamente lo que detectó validate-page.

Comandos de edición

ComandoQué hace
add-page --name "Nueva Página"Crea una página nueva en el reporte.
delete-page --page "Nombre"Elimina una página.
rename-page --page "Actual" --name "Nuevo"Renombra una página.
delete-visual --page "Nombre" --visual "id"Elimina un visual específico.
move-visual --page "P" --visual "id" --x N --y NMueve o redimensiona un visual.
10

Solución de problemas frecuentes

Síntoma Causa probable Solución
py: command not found Python no se agregó al PATH durante la instalación Reinstalar Python marcando "Add Python to PATH"
"la ejecución de scripts está deshabilitada" PowerShell tiene restricciones por defecto Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
El MCP no conecta a Power BI Ruta incorrecta en claude_desktop_config.json o PBI no está abierto Verificar la ruta exacta del ejecutable; asegurarse de que PBI tiene un reporte abierto
Visual Deneb vacío en PBI GUIDs del proyecto anterior reutilizados Correr .\pbi.ps1 detect-custom para obtener los GUIDs reales de este reporte
Navara aparece vacío / "importa el visual" .pbiviz no importado en este reporte PBI → Inicio → Más objetos visuales → Importar desde archivo → seleccionar el .pbiviz
Claude no activa el modo dashboard Skills no instaladas o carpeta incorrecta Verificar que los 4 SKILL.md están en la carpeta correcta de Claude Desktop y reiniciar la app
"Página 'X' no encontrada" El nombre de la página en el spec no coincide exactamente Correr .\pbi.ps1 list-pages y copiar el nombre exacto (diferencia mayúsculas)
Los cambios se pierden al abrir PBI Power BI estaba abierto cuando se ejecutó build-dashboard Siempre usar pbi-refresh en lugar de pbi.ps1 build-dashboard directamente
Espacio vacío al fondo de una tarjeta HTML Falta html,body{height:100%;overflow:hidden} en el CSS Agregar esa línea como primera línea del CSS en la medida DAX
Error "ModuleNotFoundError: pbir_layout" Los 3 archivos Python no están en la misma carpeta Verificar que pbir_tool.py, pbir_templates.py y pbir_layout.py están juntos en C:\Claude\pbi-motor\
Consejo de flujo de trabajo

El ciclo de trabajo normal es:

  1. Abre Power BI → crea/edita medidas con Claude via MCP (PBI abierto)
  2. Cuando las medidas estén listas, Claude cierra PBI automáticamente, construye los visuales y vuelve a abrir PBI
  3. Revisas en PBI, describes los ajustes en lenguaje natural → Claude los hace
  4. Repites hasta que el dashboard quede bien

Si algo sale mal, usa .\pbi.ps1 validate-page --page "Nombre" para detectar el problema, y fix-page para corregirlo automáticamente.
